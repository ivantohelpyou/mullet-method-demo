How It Works

Instead of hardcoded routes, everything is resolved through database lookups

Dynamic Routing

URL paths like /corporate/about are resolved through database queries, not hardcoded Flask routes.

site = Site.query.filter_by(path_prefix='corporate')

Theme System

Each site uses completely different templates and styling, all managed through database configuration.

template = f"themes/{site.theme}/page.html"

Content Management

All content, navigation, and configuration stored in database. No hardcoded content anywhere.

blocks = ContentBlock.query.filter_by(page_id=page.id)

Architecture Benefits

Why choose database-driven routing over traditional hardcoded routes?

Scalability

Add new sites without code changes. Just insert database records and create templates.

Flexibility

Change site structure, navigation, and content through database updates, not code deployments.

Multi-Tenancy

Serve multiple clients or brands from a single application with complete customization.

AI-Friendly

Perfect for AI agents to extend and customize without touching core application code.
